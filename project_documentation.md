# وثيقة مشروع منصة "ماذا" (Maza Platform)

## 1. نظرة عامة ورؤية المشروع
**منصة ماذا** هي منصة رقمية متخصصة ومبتكرة لدعم أسر ذوي الاحتياجات الخاصة، تقدم محتوى علمي وعملي في مجالات (التخاطب، تنمية المهارات، وتعديل السلوك).
* **الرؤية:** أن تكون المنصة المرجع الأول والأكثر موثوقية في الوطن العربي لتقييم وترشيح المراكز المتخصصة.
* **الرسالة:** تمكين أسر ذوي الاحتياجات الخاصة من الوصول لأفضل الخدمات العلاجية، وربطهم بشبكة من المراكز المعتمدة والخبراء المتخصصين.

## 2. نموذج العمل والربح
تعتمد المنصة على تنويع مصادر الدخل واكتساب حصة سوقية قوية عبر الشراكات:
1. **جمع وتحليل البيانات:** توفير قاعدة بيانات دقيقة للأسر وتوجيههم للمراكز المتخصصة الأنسب.
2. **نظام الكوبونات (Coupons):** بيع كوبونات خصم لخدمات المراكز (مثل مركز لايف سكيلز) ومنتجات الوسائل التعليمية (مثل إيديو ماركت) عبر واجهة المنصة.
3. **نظام التقييم والترشيح (Reviews):** مساعدة الأسر في اتخاذ القرار الأمثل بناءً على تجارب حقيقية ومعايير جودة.
4. **التسويق التبادلي:** العمل كحلقة وصل بين مقدمي الخدمة والمستفيدين (Lead Generation) للمراكز.

## 3. الميزات التقنية والنظام التشغيلي
تم تصميم المنصة لتكون نظاماً متكاملاً ومعقداً (Ecosystem) يخدم (الإدارة، الأخصائي / المركز، وولي الأمر).

### أ) نظام الجلسات والمواعيد عن بُعد (Telehealth)
* **الجدولة:** نظام تقويم (Calendar) للأخصائيين لاختيار أوقات التوافر ولولي الأمر للحجز.
* **الجلسات المرئية:** غرف فيديو مدمجة (Embedded Video) ممتثلة لمعايير الأمان العالمية (HIPAA/GDPR) كبديل أسهل من الروابط الخارجية.
* **التنبيهات:** تذكير تلقائي بمواعيد الجلسات عبر (SMS/WhatsApp/Email).

### ب) نظام التقييم والتقارير المدعوم بالذكاء الاصطناعي (AI Assessment)
* **الاختبارات القياسية:** إجراء مقاييس (CARS للتوحد، Conners لفرط الحركة، IQ للذكاء، وغيرها).
* **التصحيح التلقائي:** يقوم النظام بتحليل إجابات الطفل لتحويلها لدرجات معيارية بشكل آلي.
* **التقارير والمخرجات (Auto-Report):** يقوم الذكاء الاصطناعي بكتابة مسودة تقرير تشخيصي.
* **البرنامج العلاجي الفردي (IEP):** اقتراح خطة علاجية مخصصة للطفل (بأهداف طويلة وقصيرة المدى) تلقائياً، وتخفيف عبء التوثيق على الأخصائي.

### ج) الملف الرقمي والمتابعة (Case Management & Analytics)
* إنشاء **"ملف رقمي" (Digital Case File)** متكامل لكل حالة، ليحفظ التقرير والتدريبات بشكل آمن.
* تسجيل ملاحظات وتطور الأداء بعد كل جلسة (رفق للصور/الفيديوهات للتدريبات).
* **تصور البيانات (Data Visualization):** رسم بياني يوضح تقدم الطفل للأهل بطريقة سهلة ومبسطة للمتابعة.

### د) بوابات الدفع والتسعير الديناميكي الجغرافي (Dynamic pricing)
* **تحديد الموقع تلقائياً:** عبر (IP Address).
* **داخل مصر:** عرض الأسعار بالجنيه المصري (EGP) مع توفير سبل الدفع المحلية (فوري، ميزة، محافظ إلكترونية، وطنية).
* **خارج مصر:** عرض الأسعار بالدولار الأمريكي (USD) وتوفير وسائل دفع عالمية (بطاقات ائتمان، باي بال).
* **الفواتير الإلكترونية:** إصدار فواتير آلية عند تأكيد شراء الكوبونات أو حجز الجلسات.

## 4. البنية التحتية ومتطلبات التطوير (Tech Stack)
* **نظام تطوير الواجهات والخوادم (الأساسي):** Next.js 14/15 مع React 19، باستخدام TypeScript.
* **قاعدة البيانات:** نظام قاعدة بيانات قوي وآمن PostgreSQL (عن طريق Prisma ORM) لتأمين البيانات والسجلات الطبية.
* **محرك الذكاء الاصطناعي:** الربط مع خوادم ذكاء اصطناعي (مثل Google Generative AI المدمج، أو OpenAI API).
* **الأمان:** الالتزام بتشفير البيانات الحساسة وحفظ الخصوصية (HIPAA/GDPR).

## 5. خطة التنفيذ الموصى بها (Roadmap)
* **المرحلة 1: التأسيس والبنية التحتية:** برمجة لوحات التحكم للأدوار المختلفة، وقواعد بيانات المستخدمين والمرضى.
* **المرحلة 2: نظام المحتوى والكوبونات:** ربط المراكز الشريكة وبناء متجر الكوبونات وتقييم المراكز.
* **المرحلة 3: الذكاء الاصطناعي:** تطوير وحدة الـ AI Assessments لتشخيص الحالات وإنشاء التقارير وبرامج (IEP).
* **المرحلة 4: نظام الحجز والـ Telehealth:** إطلاق نظام الدفع المبرمج جغرافياً ودمج غرف الفيديو لإدارة الجلسات بشكل مباشر عبر المنصة.
